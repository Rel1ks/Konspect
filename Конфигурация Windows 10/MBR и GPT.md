# MBR и GPT

## Введение

**MBR** (Master Boot Record) и **GPT** (GUID Partition Table) — это две различные схемы разметки дисков, определяющие способ организации данных на жёстком диске. Выбор между ними влияет на количество поддерживаемых разделов, максимальный размер диска и совместимость с системами.

---

## MBR

**Master Boot Record** — стандартная схема разметки, появившаяся в 1983 году вместе с IBM PC DOS 2.0.

### Структура

![[14550623012025_accf102caaa970ce65d217b9ae9a8e9a57caa67c.jpg]]

### Ограничения
| Параметр | Значение |
|----------|----------|
| Максимальный размер диска | **2 ТБ** |
| Количество основных разделов | **4** |
| Поддержка UEFI | Нет (только Legacy BIOS) |

### Особенности
- Для создания более 4 разделов используется **расширенный раздел**, внутри которого размещаются **логические диски**
- Широкая совместимость со старыми системами
- Данные о разделах хранятся в одном месте (риск повреждения)

---

## GPT

**GUID Partition Table** — современный стандарт, разработанный Intel в рамках спецификации UEFI.

### Структура

![[14551123012025_08fda0244b5397e030ee401fd2bea5b24f78a72b.jpg]]

### Преимущества
| Параметр | Значение |
|----------|----------|
| Максимальный размер диска | **9,4 ЗБ** (теоретически) |
| Количество разделов | **128** (в Windows) |
| Поддержка UEFI | Да |
| Контроль целостности | CRC32-контрольные суммы |

### Особенности
- Каждый раздел имеет уникальный **GUID** (Globally Unique Identifier)
- Дублирование данных о разделах (в начале и конце диска)
- Требуется **UEFI** для загрузки (на Windows)
- Не совместим с Windows XP (32-bit) и старше

---

## Вывод

| Критерий | MBR | GPT |
|----------|-----|-----|
| Максимальный размер | 2 ТБ | До 9,4 ЗБ |
| Количество разделов | 4 основных | 128 |
| Загрузка | Legacy BIOS | UEFI |
| Надёжность | Низкая (нет резерва) | Высокая (дублирование + CRC) |
| Совместимость | Старые системы | Современные системы |
