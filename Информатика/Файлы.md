## Введение

> [!abstract] Определение
> **Файл** — это именованная единица хранения данных в файловой системе. Он выступает контейнером, внутри которого размещается информация: текст, изображение, звук, программа, архив или другой тип данных.

Файл является базовым объектом любой файловой системы. Его главная задача — ==сохранить данные в целостном и понятном виде==, чтобы операционная система и программы могли эти данные находить, открывать, изменять и передавать.

В обычной работе пользователь воспринимает файл как отдельный объект: документ, фотографию, музыку, видео или архив. Для компьютера это структура данных, у которой есть имя, свойства, формат и набор служебных характеристик.

> [!quote] Ключевая идея
> Файл нужен не только для хранения информации, но и для ее **упорядочивания, идентификации и безопасной обработки**.

---

## Основные свойства файла

> [!info] Что отличает файл от просто набора данных
> Чтобы данные считались файлом, они должны быть встроены в систему хранения и получить набор атрибутов, по которым система сможет с ними работать.

К основным признакам файла относятся:

| Свойство | Описание | Пример |
|:---|:---|:---|
| **Имя** | Позволяет отличать файл от других файлов | `report`, `photo`, `music` |
| **Расширение** | Подсказывает формат и тип содержимого | `.txt`, `.jpg`, `.mp3`, `.zip` |
| **Размер** | Показывает, сколько места файл занимает на диске | `15 KB`, `2.4 MB`, `700 MB` |
| **Дата создания** | Фиксирует момент появления файла | 27.04.2026 |
| **Дата изменения** | Показывает последнее редактирование | Изменен вчера |
| **Путь** | Указывает местоположение в файловой системе | `C:\Users\student\Documents\...` |

> [!note] Важно
> Имя файла и его расширение часто воспринимаются как одно целое, но на практике это две разные части: `имя.расширение`.

## Атрибуты файла

> [!abstract] Что такое атрибуты
> **Атрибуты файла** — это свойства, по которым файловая система и операционная система определяют, как хранить, отображать и обрабатывать файл.

Атрибуты могут быть обязательными и дополнительными. Их набор зависит от используемой файловой системы и операционной системы.

К атрибутам относятся:

- **Имя файла**
- **Расширение**
- **Размер**
- **Дата создания**
- **Дата изменения**
- **Права доступа**
- **Служебные флаги**

### Примеры атрибутов в Windows

> [!example] Типичные флаги
> В Windows у файла могут встречаться специальные атрибуты, влияющие на его поведение в системе.

| Атрибут | Значение |
|:---|:---|
| **Только чтение** | Файл нельзя случайно изменить без снятия ограничения |
| **Скрытый** | Файл не показывается в обычном режиме проводника |
| **Системный** | Файл относится к работе операционной системы |
| **Архивный** | Используется при резервном копировании |

Например, в Windows имя файла фактически обязательно для работы пользователя, хотя с точки зрения абстракции файловой системы главным является сам объект хранения. Система не позволяет хранить в одном каталоге несколько файлов с одинаковым именем и одинаковым расширением.

```text
Пример:
notes.txt
notes.docx
photo.jpg
archive.zip
```

## Форматы файлов

> [!info] Что такое формат
> **Формат файла** — это способ организации данных внутри файла. Именно формат определяет, как программа должна интерпретировать содержимое.

Если открыть текстовый файл текстовым редактором, данные будут понятны человеку. Если тем же редактором открыть изображение или музыку, содержимое будет выглядеть как набор непонятных символов, потому что формат рассчитан на другую программу.

### Основные категории форматов

| Категория | Назначение | Примеры |
|:---|:---|:---|
| **Текстовые** | Хранение текстовой информации | `.txt`, `.md`, `.docx` |
| **Графические** | Хранение изображений | `.jpg`, `.png`, `.svg` |
| **Аудио** | Хранение звука | `.mp3`, `.wav`, `.flac` |
| **Видео** | Хранение видеоданных | `.mp4`, `.mkv`, `.avi` |
| **Архивы** | Упаковка и сжатие данных | `.zip`, `.rar`, `.7z` |
| **Исполняемые** | Запуск программ и сценариев | `.exe`, `.bat`, `.ps1` |

> [!tip] Практический смысл формата
> Формат нужен для того, чтобы программа понимала, **что именно находится внутри файла** и каким способом это нужно открыть или обработать.

## Расширение файла

> [!abstract] Определение
> **Расширение** — это часть имени файла после точки, которая обычно указывает на его формат или назначение.

В Windows и во многих других системах расширение играет роль визуального ориентира. Оно помогает пользователю и программам быстро распознать тип файла.

```text
document.txt   -> текстовый файл
image.png      -> графический файл
song.mp3       -> аудиофайл
backup.zip     -> архив
```

### Зачем нужны расширения

- **Помогают системе выбрать программу для открытия**
- **Упрощают пользователю распознавание типа файла**
- **Позволяют сортировать и группировать данные**
- **Участвуют в настройке ассоциаций файлов**

> [!warning] Важно понимать
> Расширение не меняет внутреннюю структуру файла само по себе. Если просто переименовать `photo.jpg` в `photo.txt`, изображение не станет текстом. Изменится только внешний признак, а не реальное содержимое.

---

## Как связаны файл, формат и расширение

> [!note] Логика связи
> Эти три понятия связаны между собой, но не совпадают полностью.

| Понятие | Что означает |
|:---|:---|
| **Файл** | Конкретный объект хранения данных |
| **Формат** | Внутренний способ записи данных |
| **Расширение** | Внешнее обозначение формата в имени |

```text
Файл:       lecture-notes.md
Имя:        lecture-notes
Расширение: .md
Формат:     текстовый markdown-документ
```

## Где используются файлы

> [!info] Файлы повсюду
> Почти любая цифровая деятельность связана с файлами: учеба, работа, программирование, обмен данными, хранение фото и резервных копий.

- [x] **Документы** — отчеты, конспекты, таблицы, презентации
- [x] **Медиа** — изображения, музыка, видео
- [x] **Программы** — исполняемые файлы и скрипты
- [x] **Архивы** — упаковка нескольких файлов в один контейнер
- [x] **Системные данные** — конфигурации, логи, библиотеки

### Примеры из повседневной работы

| Ситуация | Какой файл используется |
|:---|:---|
| Написание заметок в Obsidian | `.md` |
| Отправка документа преподавателю | `.docx` или `.pdf` |
| Сохранение изображения | `.png` или `.jpg` |
| Передача набора материалов | `.zip` |
| Прослушивание музыки | `.mp3` |

## Почему важны правильные форматы

> [!quote] Практический вывод
> Формат и расширение нужны не для красоты, а для того, чтобы данные **открывались корректно, не терялись и обрабатывались нужной программой**.

Если формат определен неправильно или расширение вводит в заблуждение, возникают типичные проблемы:

- программа не может открыть файл;
- файл открывается не тем приложением;
- данные отображаются некорректно;
- пользователь не понимает, что именно находится внутри.

## Вывод

> [!abstract] Краткий итог
> **Файл** — это информация на компьютере, оформленная как самостоятельный объект хранения. У файла есть имя, часто есть расширение, а также набор атрибутов: размер, даты, свойства и права доступа.
>
> **Формат файла** определяет, как устроены данные внутри, а **расширение** помогает быстро понять, что это за тип данных снаружи.
>
> Без файлов невозможно организованное хранение информации, а без форматов и расширений программы не смогли бы правильно работать с этими данными.
