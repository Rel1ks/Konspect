# Модули в Python

**Модуль** — это файл с расширением `.py`, содержащий код (функции, классы, переменные). Модули позволяют организовывать и переиспользовать код.

## 1. Импорт модулей

| Синтаксис | Описание | Пример |
|---|---|---|
| `import module` | Импорт всего модуля | `import math` |
| `import module as alias` | Импорт с псевдонимом | `import numpy as np` |
| `from module import name` | Импорт конкретного имени | `from math import sqrt` |
| `from module import *` | Импорт всего (не рекомендуется) | `from math import *` |
| `from module import name as alias` | Импорт с переименованием | `from math import sqrt as sq` |

```python
import math
print(math.pi)
print(math.sqrt(16))

from math import pi, sqrt
print(pi)
print(sqrt(25))

import math as m
print(m.floor(3.7))
```

## 2. Поиск модулей

Python ищет модули в следующем порядке:

1. Текущая директория
2. `PYTHONPATH` (переменная окружения)
3. Стандартные пути установки Python (`sys.path`)

```python
import sys
for path in sys.path:
    print(path)
```

## 3. Повторный импорт

Модуль импортируется **один раз**. Повторный импорт использует кеш.

```python
import sys
# список уже загруженных модулей
print(sys.modules)

# принудительная перезагрузка
import importlib
importlib.reload(math)
```

## 4. `if __name__ == "__main__"`

Позволяет выполнять код только при запуске файла напрямую, а не при импорте.

```python
# my_module.py
def greet(name):
    return f"Привет, {name}!"

if __name__ == "__main__":
    # этот код выполнится только при запуске файла
    print(greet("Анна"))
```

```python
# main.py
import my_module  # greet доступна, но print НЕ выполнится
```

## 5. Пакеты (папки с модулями)

**Пакет** — папка, содержащая файл `__init__.py` (может быть пустым).

```
my_package/
├── __init__.py
├── module_a.py
└── sub_package/
    ├── __init__.py
    └── module_b.py
```

```python
import my_package.module_a
from my_package.sub_package import module_b

# или с длинными именами
from my_package.module_a import some_function
```

## 6. `__init__.py`

Файл `__init__.py` позволяет настроить поведение пакета.

```python
# __init__.py
from .module_a import some_function
from .sub_package.module_b import another_function

__all__ = ["some_function", "another_function"]
```

## 7. Относительный импорт

```python
# внутри пакета
from . import module_a       # текущий пакет
from .module_a import func   # из модуля в той же папке
from .. import other_module  # из родительского пакета
from ..sub import module_c   # из другого подпакета
```

## 8. Создание своего модуля

```python
# utils.py
def add(a, b):
    return a + b

def multiply(a, b):
    return a * b

PI = 3.14159
```

```python
# main.py
import utils

print(utils.add(5, 3))
print(utils.PI)
```

## 9. `__all__`

Контролирует, что будет импортировано при `from module import *`.

```python
# module.py
__all__ = ["func_a", "CONSTANT"]  # только они будут импортированы

def func_a():
    pass

def func_b():  # не попадёт при import *
    pass
```
