# Хеш-таблицы (Hash Tables) в Python

**Хеш-таблица** — структура данных, которая хранит пары «ключ — значение» и обеспечивает доступ к значению по ключу за O(1) в среднем.

## 1. Принцип работы

```
Ключ → [Хеш-функция] → Хеш (число) → Индекс в массиве → Значение
```

```python
# упрощённая схема:
# key = "name"
# hash("name") = 123456
# index = hash("name") % array_size  # 123456 % 10 = 6
# array[6] = ("name", "Анна")
```

## 2. Словарь (dict) — реализация хеш-таблицы в Python

```python
d = {}
d["name"] = "Анна"    # вставка — O(1)
x = d["name"]          # получение — O(1)
del d["name"]          # удаление — O(1)
"name" in d            # проверка — O(1)
```

### Основные операции

```python
phonebook = {
    "Анна": "123-456",
    "Борис": "789-012",
    "Вика": "345-678"
}

# вставка/обновление
phonebook["Глеб"] = "901-234"

# получение
print(phonebook["Анна"])
print(phonebook.get("Дима", "не найден"))

# удаление
del phonebook["Борис"]
phonebook.pop("Вика")

# проверка
if "Анна" in phonebook:
    print("Анна в списке")

# длина
print(len(phonebook))

# обход
for name, phone in phonebook.items():
    print(f"{name}: {phone}")
```

## 3. Хеш-функция в Python

```python
# встроенная hash()
print(hash("hello"))    # число (меняется между запусками)
print(hash(42))         # 42 (int хешируется как само число)
print(hash((1, 2, 3)))  # кортеж можно

# hash([1, 2, 3])  # TypeError — список изменяемый, нехешируемый
```

### Требования к хешируемым ключам

| Можно (хешируемые) | Нельзя (нехешируемые) |
|---|---|
| `int`, `float`, `str`, `bool` | `list`, `dict`, `set` |
| `tuple` (если внутри хешируемое) | `tuple` с изменяемыми внутри |
| `frozenset` | |

```python
# можно
d = {
    1: "int",
    3.14: "float",
    "str": "string",
    (1, 2): "tuple",
    True: "bool",
}

# нельзя
# d[[1, 2]] = "list"  # TypeError
# d[{"a": 1}] = "dict"  # TypeError
```

## 4. Коллизии

Разные ключи могут дать одинаковый хеш (или индекс). Python использует **открытую адресацию** с псевдослучайным пробированием.

```python
# упрощённый пример коллизии — когда хеш % size совпадает
# Python решает это внутри, мы этого не видим
d = {}
for i in range(1000):
    d[i] = i * 2  # все работает, коллизии скрыты
```

### Способы решения коллизий

| Метод | Описание |
|---|---|
| **Цепочки** (chaining) | В каждой ячейке — список элементов |
| **Открытая адресация** | Ищет следующую свободную ячейку |
| **Двойное хеширование** | Второй хеш для шага пробирования |

Python использует открытую адресацию.

## 5. Сложность операций

| Операция | Средняя | Худшая (редко) |
|---|---|---|
| Вставка | O(1) | O(n) |
| Получение | O(1) | O(n) |
| Удаление | O(1) | O(n) |
| Поиск ключа | O(1) | O(n) |
| Обход | O(n) | O(n) |

Худший случай (O(n)) возникает при множестве коллизий.

## 6. Load factor (коэффициент загрузки)

```
load_factor = количество элементов / размер таблицы
```

- Python автоматически увеличивает размер таблицы (rehash), когда load_factor превышает порог (~2/3)
- Все элементы перехешируются и распределяются заново — O(n), но редко

## 7. Своя хеш-таблица (учебная реализация)

```python
class HashTable:
    def __init__(self, size=10):
        self.size = size
        self.table = [[] for _ in range(size)]
        self.count = 0

    def _hash(self, key):
        return hash(key) % self.size

    def _resize(self):
        """Увеличить размер и перехешировать"""
        old_table = self.table
        self.size *= 2
        self.table = [[] for _ in range(self.size)]
        self.count = 0

        for bucket in old_table:
            for key, value in bucket:
                self[key] = value

    def __setitem__(self, key, value):
        index = self._hash(key)
        bucket = self.table[index]

        for i, (k, v) in enumerate(bucket):
            if k == key:
                bucket[i] = (key, value)
                return

        bucket.append((key, value))
        self.count += 1

        if self.count / self.size > 0.7:
            self._resize()

    def __getitem__(self, key):
        index = self._hash(key)
        bucket = self.table[index]

        for k, v in bucket:
            if k == key:
                return v

        raise KeyError(key)

    def __delitem__(self, key):
        index = self._hash(key)
        bucket = self.table[index]

        for i, (k, v) in enumerate(bucket):
            if k == key:
                bucket.pop(i)
                self.count -= 1
                return

        raise KeyError(key)

    def __contains__(self, key):
        try:
            self[key]
            return True
        except KeyError:
            return False

    def __len__(self):
        return self.count

    def __repr__(self):
        pairs = []
        for bucket in self.table:
            for k, v in bucket:
                pairs.append(f"{k}: {v}")
        return "{" + ", ".join(pairs) + "}"

# использование
ht = HashTable()
ht["name"] = "Анна"
ht["age"] = 25
ht["city"] = "Москва"

print(ht["name"])
print(ht)
print("age" in ht)
print(len(ht))
```

## 8. Практические примеры

### Подсчёт частоты

```python
# через dict
text = "abracadabra"
freq = {}
for ch in text:
    freq[ch] = freq.get(ch, 0) + 1
print(freq)

from collections import Counter

```

### Группировка

```python
words = ["яблоко", "груша", "арбуз", "ананас", "абрикос"]
by_first = {}

for word in words:
    first = word[0]
    if first not in by_first:
        by_first[first] = []
    by_first[first].append(word)

print(by_first)
```

### Кеш (мемоизация)

```python
cache = {}

def fib(n):
    if n in cache:
        return cache[n]
    if n <= 1:
        return n
    result = fib(n - 1) + fib(n - 2)
    cache[n] = result
    return result

print(fib(50))
```

### Поиск дубликатов

```python
def has_duplicates(nums):
    seen = {}
    for num in nums:
        if num in seen:
            return True
        seen[num] = True
    return False

print(has_duplicates([1, 2, 3, 4]))    # False
print(has_duplicates([1, 2, 2, 3]))    # True
```

### Два сумма (Two Sum)

```python
def two_sum(nums, target):
    """Найти индексы двух чисел, дающих target"""
    seen = {}
    for i, num in enumerate(nums):
        complement = target - num
        if complement in seen:
            return [seen[complement], i]
        seen[num] = i
    return []

print(two_sum([2, 7, 11, 15], 9))  # [0, 1]
```

## 9. Сравнение с другими структурами

| Структура | Поиск | Вставка | Удаление | Упорядоченность |
|---|---|---|---|---|
| **Хеш-таблица (dict)** | O(1) | O(1) | O(1) | Нет (с 3.7 — порядок вставки) |
| **Список (list)** | O(n) | O(1)/O(n) | O(1)/O(n) | Да |
| **Бинарное дерево (TreeMap)** | O(log n) | O(log n) | O(log n) | Да (по ключу) |
| **Множество (set)** | O(1) | O(1) | O(1) | Нет |

## 10. Именованные хеш-таблицы

```python
from collections import defaultdict, OrderedDict, Counter

# defaultdict — значение по умолчанию для отсутствующего ключа
d = defaultdict(list)
d["a"].append(1)
d["a"].append(2)
print(dict(d))

# OrderedDict — гарантирует порядок (был важен до Python 3.7)
from collections import OrderedDict
od = OrderedDict()
od["a"] = 1
od["b"] = 2

# Counter — подсчёт
c = Counter("abracadabra")
print(c.most_common(2))
```
