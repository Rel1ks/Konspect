## Полный список встроенных исключений

Python содержит обширную иерархию встроенных исключений, которые покрывают большинство типовых ошибок:
## Системные исключения

- **BaseException** — базовое исключение, от которого наследуются все остальные
    
- **SystemExit** — порождается функцией sys.exit() при выходе из программы
    
- **KeyboardInterrupt** — возникает при прерывании программы пользователем (Ctrl+C)
    
- **GeneratorExit** — возникает при вызове метода close() объекта-генератора
    

## Часто используемые исключения

- **Exception** — базовый класс для большинства обычных исключений
    
- **TypeError** — операция применяется к объекту неподходящего типа
    
- **ValueError** — функция получает аргумент правильного типа, но с некорректным значением
    
- **ZeroDivisionError** — деление на ноль
    
- **IndexError** — индекс выходит за пределы последовательности
    
- **KeyError** — ключ отсутствует в словаре
    
- **AttributeError** — отсутствует атрибут объекта
    
- **NameError** — переменная не определена
    
- **FileNotFoundError** — файл не найден
    
- **ImportError** — ошибка импорта модуля
    
- **RuntimeError** — общая ошибка времени выполнения
    

## Расширенная обработка исключений

## Множественные блоки except

Можно обрабатывать разные типы исключений по-разному:

```python
try:
    # код с потенциальными ошибками
    result = int("abc")
except TypeError:
    print("Обнаружена ошибка TypeError")
except ValueError:
    print("Обнаружена ошибка ValueError")
except Exception:
    print("Что-то пошло не так")

```

## Получение информации об исключении

Можно получить объект исключения для детального анализа:​

```python
try:
    x = 1 / 0
except ZeroDivisionError as e:
    print(f"Поймано исключение: {e}")

```

## Блоки else и finally

- **else** — выполняется, если в блоке try не возникло исключений
    
- **finally** — выполняется всегда, независимо от наличия исключений (для освобождения ресурсов)

```python
try:
    file = open('data.txt', 'r')
    content = file.read()
except FileNotFoundError:
    print("Файл не найден")
else:
    print("Файл успешно прочитан")
finally:
    file.close()  # выполнится в любом случае

```

## Генерация исключений с raise

## Основное использование raise

Инструкция **raise** позволяет принудительно вызвать исключение в любом месте кода:

```python
raise Exception('Какой-то текст')
raise ValueError("Несоответствующее значение")
```


## Повторный вызов исключения

Можно использовать **raise** без аргументов внутри блока except для повторного вызова перехваченного исключения:

```python
try:
    print('1' + 1)
except:
    print("Произошла ошибка")
    raise  # повторно вызывает TypeError

```

## Цепочка исключений

Если внутри блока except или finally возникает новое исключение, предыдущее исключение автоматически привязывается к нему через атрибут `__context__`.​

## Использование assert

Инструкция **assert** проверяет условие и вызывает исключение **AssertionError**, если условие ложно:​

```python
try:
    assert 2 + 2 == 4  # пройдет
    assert 1 + 2 == 4  # вызовет AssertionError
except AssertionError:
    print("Утверждение ложно")

```

## Создание пользовательских исключений

Для специфичных задач можно создавать собственные классы исключений, наследуясь от Exception:​

```python
class CustomError(Exception):
    def __init__(self, message, code):
        super().__init__(message)
        self.code = code

try:
    raise CustomError("Специфичная ошибка", 404)
except CustomError as e:
    print(f"Код ошибки: {e.code}, Сообщение: {e}")

```
