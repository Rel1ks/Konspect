# Конкурентность (Concurrency) в Python

**Конкурентность** — выполнение нескольких задач одновременно (или с переключением между ними).

## 1. Основные понятия

| Термин | Описание |
|---|---|
| **Конкурентность** | Несколько задач прогрессируют (переключение) |
| **Параллелизм** | Несколько задач выполняются физически одновременно |
| **GIL** | Global Interpreter Lock — блокировка CPython |
| **Блокирующий I/O** | Ожидание ввода/вывода (сеть, диск) |
| **CPU-bound** | Загрузка процессора (вычисления) |
| **I/O-bound** | Ожидание ввода/вывода |

## 2. Модели конкурентности в Python

| Модель | Процессы | Потоки | GIL | I/O-bound | CPU-bound |
|---|---|---|---|---|---|
| **threading** | 1 | Много | Да | ✅ | ❌ |
| **multiprocessing** | Много | — | Нет | ✅ | ✅ |
| **asyncio** | 1 | 1 | Да | ✅ | ❌ |
| **concurrent.futures** | Оба | Оба | Зависит | ✅ | ✅ |

### Когда что использовать

```python
# I/O-bound (сеть, диск, БД) → asyncio или threading
# CPU-bound (вычисления) → multiprocessing
# Смешанное → concurrent.futures
```

## 3. threading — потоки

Потоки в одном процессе. Делят память. Под GIL.

```python
import threading
import time

def worker(name):
    print(f"Поток {name} начал")
    time.sleep(1)
    print(f"Поток {name} завершил")

threads = []
for i in range(3):
    t = threading.Thread(target=worker, args=(i,))
    t.start()
    threads.append(t)

for t in threads:
    t.join()

print("Все потоки завершены")
```

### ThreadPoolExecutor

```python
from concurrent.futures import ThreadPoolExecutor
import time

def fetch_url(url):
    time.sleep(1)
    return f"Data from {url}"

urls = [f"url{i}" for i in range(10)]

with ThreadPoolExecutor(max_workers=5) as executor:
    results = list(executor.map(fetch_url, urls))

print(results)
```

## 4. multiprocessing — процессы

Каждый процесс — отдельный интерпретатор. Нет GIL.

```python
import multiprocessing
import time

def worker(name):
    print(f"Процесс {name} начал")
    time.sleep(1)
    print(f"Процесс {name} завершил")

if __name__ == "__main__":
    processes = []
    for i in range(3):
        p = multiprocessing.Process(target=worker, args=(i,))
        p.start()
        processes.append(p)

    for p in processes:
        p.join()
```

### Pool

```python
from multiprocessing import Pool

def square(x):
    return x * x

if __name__ == "__main__":
    with Pool(4) as pool:
        results = pool.map(square, range(10))
        print(results)
```

## 5. asyncio — асинхронность

Один поток, одно ядро, кооперативная многозадачность.

```python
import asyncio

async def fetch(url):
    print(f"Загрузка {url}")
    await asyncio.sleep(1)
    return f"Data from {url}"

async def main():
    tasks = [fetch(f"url{i}") for i in range(3)]
    results = await asyncio.gather(*tasks)
    print(results)

asyncio.run(main())
```

## 6. concurrent.futures

Единый интерфейс для потоков и процессов.

```python
from concurrent.futures import ThreadPoolExecutor, ProcessPoolExecutor
import time

def task(n):
    time.sleep(1)
    return n * 2

# потоки
with ThreadPoolExecutor(max_workers=4) as executor:
    futures = [executor.submit(task, i) for i in range(10)]
    results = [f.result() for f in futures]

# процессы
with ProcessPoolExecutor(max_workers=4) as executor:
    results = list(executor.map(task, range(10)))
```

## 7. I/O-bound vs CPU-bound

### I/O-bound — threading или asyncio

```python
# threading
import threading
import requests

def fetch(url):
    response = requests.get(url)
    print(f"{url}: {len(response.content)}")

urls = ["https://example.com"] * 10
threads = [threading.Thread(target=fetch, args=(u,)) for u in urls]
for t in threads: t.start()
for t in threads: t.join()

# asyncio
import asyncio
import aiohttp

async def fetch(session, url):
    async with session.get(url) as response:
        return len(await response.read())

async def main():
    async with aiohttp.ClientSession() as session:
        tasks = [fetch(session, f"https://example.com") for _ in range(10)]
        results = await asyncio.gather(*tasks)
        print(results)

asyncio.run(main())
```

### CPU-bound — multiprocessing

```python
from multiprocessing import Pool

def fibonacci(n):
    a, b = 0, 1
    for _ in range(n):
        a, b = b, a + b
    return a

if __name__ == "__main__":
    with Pool(4) as pool:
        results = pool.map(fibonacci, [100000] * 8)
        print(results)
```

## 8. GIL (Global Interpreter Lock)

**GIL** — блокировка CPython, разрешающая выполнение только одного потока в один момент времени.

```python
import threading
import time

def count(n):
    while n > 0:
        n -= 1

# CPU-bound — потоки не ускоряют из-за GIL
def test_threading():
    start = time.time()
    threads = [threading.Thread(target=count, args=(50000000,)) for _ in range(2)]
    for t in threads: t.start()
    for t in threads: t.join()
    print(f"Threading: {time.time() - start:.2f} сек")

# Аналог в один поток
def test_single():
    start = time.time()
    count(50000000)
    count(50000000)
    print(f"Single: {time.time() - start:.2f} сек")

# процессы — ускоряют (нет GIL)
def test_multiprocessing():
    from multiprocessing import Process
    start = time.time()
    processes = [Process(target=count, args=(50000000,)) for _ in range(2)]
    for p in processes: p.start()
    for p in processes: p.join()
    print(f"Processes: {time.time() - start:.2f} сек")
```

## 9. Синхронизация

### Lock

```python
import threading

lock = threading.Lock()
counter = 0

def increment():
    global counter
    for _ in range(100000):
        with lock:
            counter += 1

threads = [threading.Thread(target=increment) for _ in range(5)]
for t in threads: t.start()
for t in threads: t.join()
print(counter)
```

### Queue

```python
from queue import Queue
import threading

queue = Queue()

def producer():
    for i in range(10):
        queue.put(i)

def consumer():
    while True:
        item = queue.get()
        if item is None:
            break
        print(f"Получено: {item}")
        queue.task_done()

t_prod = threading.Thread(target=producer)
t_cons = threading.Thread(target=consumer)
t_prod.start(); t_cons.start()
t_prod.join()
queue.put(None)
t_cons.join()
```

## 10. Сравнение

| | threading | multiprocessing | asyncio |
|---|---|---|---|
| GIL | Есть | Нет | Есть |
| Память | Общая | Раздельная | Общая |
| Создание | Лёгкое | Тяжёлое | Лёгкое |
| I/O-bound | ✅ | ✅ | ✅ |
| CPU-bound | ❌ | ✅ | ❌ |
| Количество | ~1000 | ~CPU cores | ~10000+ |
| Синтаксис | Сложный | Сложный | async/await |
